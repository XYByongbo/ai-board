# 部署教程（阿里云大陆 ECS · 已备案 · Ubuntu）

把这个纯前端项目部署到你自己的阿里云服务器，并用你在阿里云买的域名访问。

> **本质**：本项目 `pnpm build` 后只产出 `dist/` 下 3 个静态文件（`index.html` + 一个 js + 一个 css）。
> 部署 = 把 `dist/` 传到服务器 → Nginx 托管 → 域名解析到服务器 IP → （可选）加 HTTPS。
> 全程约 30 分钟，以后发新版本只需 1 条命令。

---

## 0. 前置准备（先把这几样东西备齐）

**本地（你的 Mac）需要：**
- 已能跑 `pnpm build`（项目用 pnpm；没装就 `npm i -g pnpm`）。
- `ssh`、`rsync`：macOS 自带，无需安装。

**服务器需要：**
- 阿里云 ECS 的【公网 IP】（控制台实例详情里能看到）。
- 能 SSH 登录：默认用户 `root`，买服务器时设的密码（或密钥）。

**信息清单（先填好，后面到处要用）：**

| 项 | 你的值 | 示例 |
|---|---|---|
| 公网 IP | `__________` | `1.2.3.4` |
| 域名 | `__________` | `board.example.com` 或 `example.com` |
| 站点目录 | `/var/www/ai-board` | 默认即可，别改 |

> ⚠️ **关于备案**：你说已备案。请确认这个备案是“接入到这台阿里云 ECS”的（在阿里云买的域名 + 阿里云服务器，通常就是）。
> 否则大陆服务器的 **80 端口会被阿里云拦截**，表现为「域名 ping 得通、但网页打不开」。

---

## 1. 域名解析：把域名指向服务器 IP

1. 登录 [阿里云控制台](https://dns.console.aliyun.com/) → **域名** → 找到你的域名 → 点 **解析 / 解析设置**。
2. 点 **添加记录**，按下表填：

   | 字段 | 填什么 | 说明 |
   |---|---|---|
   | 记录类型 | `A` | 把域名指向一个 IP |
   | 主机记录 | `@`（用主域名）或 `board`（用子域名 `board.你的域名`） | 见下方说明 |
   | 解析线路 | 默认 | 不用动 |
   | 记录值 | 你的**公网 IP** | 例如 `1.2.3.4` |
   | TTL | 10 分钟 | 默认即可 |

   - **想用主域名** `example.com` → 主机记录填 `@`；通常再加一条主机记录 `www` 指同一 IP。
   - **想用子域名** `board.example.com` → 主机记录填 `board`。
3. 保存。已备案的情况下，通常几分钟内生效。

**验证解析是否生效**（在你的 Mac 终端）：
```bash
ping 你的域名        # 看到的 IP == 你的公网 IP 就对了，Ctrl+C 停止
```

---

## 2. 放行端口：ECS 安全组开放 80 / 443

> 🔥 **新手最常踩的坑**：服务器装好了、解析也对了，但网页就是打不开——99% 是这一步没做。
> 阿里云 ECS 默认**不放行** 80/443，必须手动开。

1. 阿里云控制台 → **云服务器 ECS** → **实例** → 点你的实例 → **安全组** → **配置规则**。
2. **入方向** → **手动添加**，加两条：

   | 端口范围 | 协议 | 授权对象（源） | 用途 |
   |---|---|---|---|
   | `80/80` | TCP | `0.0.0.0/0` | HTTP |
   | `443/443` | TCP | `0.0.0.0/0` | HTTPS |

3. 保存。（22 端口 SSH 通常默认已放行，否则你也连不上服务器。）

---

## 3. 服务器安装 Nginx

SSH 登录服务器（在你的 Mac 终端）：
```bash
ssh root@你的公网IP        # 首次会问 yes/no，输 yes；再输密码
```

登录后执行（Ubuntu / Debian）：
```bash
apt update
apt install -y nginx
systemctl enable --now nginx     # 启动并设为开机自启
```

**验证**：浏览器访问 `http://你的公网IP`，看到 Nginx 默认欢迎页（"Welcome to nginx!"）就成功了。
> 如果这一步打不开，回去检查第 2 步的安全组。

---

## 4. 配置 Nginx 站点

本项目自带配置模板 `deploy/nginx.conf`，把它放到服务器并改一处域名即可。

**做法 A（推荐，从本地传上去）** — 在你的 Mac、项目根目录执行：
```bash
scp deploy/nginx.conf root@你的公网IP:/etc/nginx/conf.d/ai-board.conf
```

**做法 B（直接在服务器上创建）** — 在服务器执行 `nano /etc/nginx/conf.d/ai-board.conf`，粘贴 `deploy/nginx.conf` 的内容。

然后**在服务器上**改域名并重载：
```bash
# 把配置里的 your-domain.com 换成你的真实域名
sed -i 's/your-domain.com/你的域名/g' /etc/nginx/conf.d/ai-board.conf

nginx -t                  # 测试配置语法，必须看到 "syntax is ok" + "test is successful"
systemctl reload nginx    # 重载生效
```

> 这个配置做了什么：站点根目录设为 `/var/www/ai-board`；`/assets/`（带哈希的 js/css）长缓存、`index.html` 不缓存（保证你更新后用户立刻看到新版）。
> 配置里的 `root` 路径必须和第 5 步上传的目标目录一致，默认都是 `/var/www/ai-board`，别改乱。

---

## 5. 本地构建并上传 dist（一键脚本）

本项目自带 `deploy/deploy.sh`：自动 `pnpm build` + 用 rsync 把 `dist/` 同步到服务器。

**先改脚本顶部 2 个变量**，打开 `deploy/deploy.sh`：
```bash
SERVER="root@你的公网IP"          # 例如 root@1.2.3.4
REMOTE_DIR="/var/www/ai-board"   # 保持默认，和 nginx 的 root 一致
```

**然后在项目根目录执行**：
```bash
bash deploy/deploy.sh
```

脚本会：构建 → 在服务器建好 `/var/www/ai-board` → 把 `dist/` 增量同步上去（`--delete` 会清掉服务器上的旧文件，保持干净）。

**验证**：浏览器访问 `http://你的域名`，应该能看到 Token 看板页面了 🎉

> 💡 **少输密码**：每次 deploy 都要输服务器密码很烦。配一次 SSH 免密即可：
> ```bash
> ssh-keygen -t ed25519        # 一路回车（本地已有密钥可跳过）
> ssh-copy-id root@你的公网IP   # 输一次密码，之后永久免密
> ```

---

## 6. 开启 HTTPS（让地址栏变成 🔒 https）

免费证书，几条命令搞定。**在服务器上执行**：
```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d 你的域名
```

certbot 会交互式询问：
- **邮箱**：填你的邮箱（证书到期前会邮件提醒）。
- **同意条款**：输 `Y`。
- **是否把 HTTP 跳转到 HTTPS**：选 **2（Redirect，强制跳 https，推荐）**。

完成后它会自动改写 `ai-board.conf`、加上 443 配置和证书，无需你手动改。

**验证 & 自动续期**：
```bash
certbot renew --dry-run         # 测试续期流程，不报错即可（证书 90 天，会自动续）
```
现在访问 `https://你的域名`，地址栏带 🔒。

---

## 7. 部署完成验证清单

- [ ] `ping 你的域名` 返回的是你的公网 IP
- [ ] `http://公网IP` 能看到 Nginx 欢迎页（或部署后看到项目页）
- [ ] `http://你的域名` 能打开 Token 看板
- [ ] `https://你的域名` 带锁、且 http 自动跳 https
- [ ] 在页面里输入你的 API Key，能正常查出数据（接口已开 CORS，纯前端直连）

---

## 8. 以后怎么发新版本？

改完代码后，在本地项目根目录**只需一条命令**：
```bash
bash deploy/deploy.sh
```
它会重新构建并同步到服务器，刷新域名即是最新版（`index.html` 不缓存，更新即时生效）。

## 9. 打不开时的排查顺序（按这个顺序查）

| 现象 | 大概率原因 | 怎么查 / 怎么修 |
|---|---|---|
| `ping 域名` 不通 / IP 不对 | 解析没生效或填错 | 回第 1 步检查 A 记录、记录值；等几分钟 |
| `http://公网IP` 打不开 | 安全组没放行 / Nginx 没起 | 第 2 步放行 80；服务器上 `systemctl status nginx` |
| IP 能开、**域名**打不开 | 备案接入不对（大陆）或解析没好 | 确认备案接入到本服务器；`ping 域名` 看 IP |
| 域名打开是 Nginx 欢迎页，不是项目 | 站点配置没匹配 / dist 没传 | `nginx -t`；确认第 4、5 步都做了；看 `server_name` |
| 页面 404 / 403 | dist 没传到 `root` 目录 / 权限 | `ls /var/www/ai-board` 看有没有 index.html |
| 接口报错、查不出数据 | 这是你的 API Key 或网络问题 | 与部署无关；F12 看 Network 里的请求响应 |

**看 Nginx 日志**（服务器上，排查利器）：
```bash
tail -n 50 /var/log/nginx/error.log
tail -n 50 /var/log/nginx/access.log
```

> 补充：阿里云 ECS 默认用安全组控流量，系统防火墙 `ufw` 一般是关的。
> 若你手动开过 `ufw`，需放行：`ufw allow 'Nginx Full'`。
